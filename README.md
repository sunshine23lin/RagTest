# RagTest - 电力设计文档智能分析系统

基于 LangGraph 工作流 + 混合检索（BM25 + 向量语义）的电力设计文档智能问答系统，支持文档解析、理解、上下文不一致检测、复杂表格数据提取与精准问答。

---

## 一、项目简介

本项目面向电力设计研究院场景，针对工程初步设计说明书等专业文档，提供以下核心能力：

- **文档解析**：支持 PDF/DOCX 格式，自动提取文本和表格
- **混合检索**：BM25 关键词检索 + 向量语义检索，RRF 融合排序
- **智能问答**：基于 LangGraph 工作流，7 节点流水线处理
- **不一致检测**：自动检测项目名称、数据口径等上下文矛盾
- **表格理解**：复杂表格转 JSON，去重破损数据，精准提取数值

### 5.2 智能体工作流（agent_workflow.py）

工作流由 7 个节点组成，按顺序执行：

```
BM25检索 → 向量检索 → RRF融合 → 重排 → 上下文构建 → 答案生成 → 后处理验证
```

| 节点 | 方法 | 功能 |
|------|------|------|
| BM25 检索 | `_retrieve_bm25` | 基于关键词匹配检索，对精确术语、数值检索效果好 |
| 向量检索 | `_retrieve_vector` | 基于语义相似度检索，支持内存/磁盘双模式 |
| RRF 融合 | `_rrf_fusion` | 综合两路检索结果，BM25 权重 2.0，向量权重 1.0 |
| 重排 | `_rerank` | 使用 LLM 对候选结果进行语义重排，提升相关性 |
| 上下文构建 | `_build_context` | 将检索结果格式化为 LLM 可理解的上下文 |
| 答案生成 | `_generate_answer` | 调用 LLM 生成答案，支持主备模型自动切换 |
| 后处理验证 | `_verify_and_correct_answer` | 统一信息来源格式，修正文档标题 |

### 5.3 主入口（agent_main.py）

`RagAgentPipeline` 类提供完整的 RAG 管道功能：

| 方法 | 功能 |
|------|------|
| `_get_collection_name(doc_path)` | 根据文档路径 MD5 生成唯一集合名称 |
| `is_collection_exists()` | 检查 Milvus 中是否已存在该文档的索引 |
| `load_and_process_document(doc_path)` | 加载文档并切片 |
| `build_indexes(chunked_docs)` | 构建向量索引 + BM25 索引 |

## 六、核心思路

### 6.1 双路检索 + RRF 融合

- **BM25 检索**：基于关键词匹配，对精确术语、数值、型号等检索效果好
- **向量检索**：基于语义相似度，对概念理解、同义词匹配效果好
- **RRF 融合**：Reciprocal Rank Fusion 算法，综合两路结果
  - BM25 权重：2.0（关键词匹配更精确）
  - 向量权重：1.0
  - RRF 常数 k=60

### 6.2 智能上下文构建

1. **全局分析类问题**（如"检查不一致"）：
   - 跳过检索，直接使用全文压缩策略
   - 按页面分组，每页限制 3000 字符
   - 自动检测项目名称不一致

2. **普通问题**：
   - 表格数据转 JSON 格式，便于 LLM 理解
   - 检测并跳过破损的表格文本（无 Markdown 标记但包含表格数据）
   - 使用文档大标题作为统一信息来源

### 6.3 复杂表格数据读取策略

**问题**：PDF 解析时，表格可能被解析为多个版本（完整 Markdown 表格 + 破损纯文本），导致数据重复或冲突。

**解决方案**：

1. **表格独立切片**：表格单独作为一个 chunk，不与其他文本混合
2. **内容签名去重**：提取表格中的关键数据特征（如 `6633#_90m`），用于识别重复数据
3. **破损文本检测**：
   - 检查文本是否包含与表格相同的设备/高度标识
   - 检查是否包含多行纯数字数据（无 Markdown 表格标记 `|`）
   - 如果满足条件，判定为破损表格文本，直接跳过
4. **表格转 JSON**：将 Markdown 表格解析为结构化 JSON，第一列作为行名，其他列作为键值对

```python
# 表格转 JSON 示例
# 原始 Markdown 表格：
# | 测风点 | 90m | 95m |
# |--------|-----|-----|
# | 6633#  | 8.32| 8.45|

# 转换为 JSON：
[
    {"测风点": "6633#", "data": {"90m": "8.32", "95m": "8.45"}}
]
```

### 6.4 项目名称不一致自动检测

**检测逻辑**：
1. 从每页提取项目名称（支持多种正则模式）
2. 对比不同页的项目名称，标准化后分组
3. 通过重复字符检测 OCR 错误（如"上上"、"的的"、"中中"）
4. 通过出现频率判断正确名称（全文出现次数最多的为正确名称）
5. 输出正确名称和错误名称（标注错误类型：OCR 识别错误 / 笔误）

### 6.5 答案格式化与后处理

- **统一信息来源格式**：只使用文档大标题，不细化到小标题或表格内容
- **正则替换**：支持多种格式匹配（方括号、多页码等）
- **答案验证**：检查回答质量，失败时自动切换到备用模型

## 七、数据切片策略

| 策略 | 说明 |
|------|------|
| 表格独立处理 | 表格单独作为一个 chunk，不切分 |
| 文本递归切分 | 按段落、句子、字符递归切分 |
| 结构感知 | 识别标题层级，保持章节完整性 |
| 重叠上下文 | chunk_size=500, chunk_overlap=50，保持语义连贯性 |

## 九、快速开始

### 9.1 前置要求

1. **启动 Milvus 向量数据库**
   ```bash
   docker run -d --name milvus -p 19530:19530 milvusdb/milvus:latest
   ```

2. **安装 Python 依赖**
   ```bash
   pip install -r requirements.txt
   ```

3. **配置 API Key**
   
   在 `config.py` 中设置智谱 API Key，或通过环境变量：
   ```bash
   set ZHIPU_API_KEY=your_api_key_here
   ```

### 9.2 命令行使用

```bash
# 首次运行（自动构建索引）
python agent_main.py --doc_path "你的文档路径.pdf" --batch

# 再次运行（直接使用已有索引，不会重新分片）
python agent_main.py --doc_path "你的文档路径.pdf" --batch

# 单个问题问答
python agent_main.py --doc_path "你的文档路径.pdf" --query "本工程项目名称是什么？"

# 强制重建索引
python agent_main.py --doc_path "你的文档路径.pdf" --rebuild --batch

# 清空索引
python agent_main.py --doc_path "你的文档路径.pdf" --clear

# 交互式对话模式
python agent_main.py --doc_path "你的文档路径.pdf"

# 使用指定问题文件（JSON 格式）
python agent_main.py --doc_path "你的文档路径.pdf" --questions_file questions.json
```

### 9.3 Python API 使用

```python
from agent_main import RagAgentPipeline

# 初始化管道（use_memory_search=False 使用 Milvus 查询）
pipeline = RagAgentPipeline(doc_path="你的文档路径.pdf", use_memory_search=False)

# 首次运行：加载文档并构建索引
chunked_docs = pipeline.load_and_process_document("你的文档路径.pdf")
pipeline.build_indexes(chunked_docs)

# 初始化智能体
pipeline.init_agent()

# 单个问题
result = pipeline.agent.ask("本工程项目名称是什么？")
print(result['answer'])

# 批量问题
questions = [
    "本工程项目名称是什么？",
    "本项目风机的单机容量有哪些？",
    "本项目的总装机容量是多少？"
]
results = pipeline.agent.ask_batch(questions)
for r in results:
    print(f"问题: {r['question']}")
    print(f"回答: {r['answer']}\n")
```

```bash
python agent_main.py --doc_path "A.pdf" --batch

# 文档 B（自动创建集合 rag_yyy，与 A 独立）
python agent_main.py --doc_path "B.pdf" --batch

# 切换回文档 A（直接使用已有集合，不会重新分片）
python agent_main.py --doc_path "A.pdf" --batch
```

## 十、配置说明

在 `config.py` 中可以修改以下配置：

| 配置项 | 说明 |
|--------|------|
| `ACTIVE_LLM_PROVIDER` | 当前使用的 LLM 提供商（dashscope/openrouter/zhipu） |
| `LLM_PROVIDERS` | 各 LLM 提供商的 API Key、模型、温度等配置 |
| `MILVUS_CONFIG` | Milvus 连接参数和索引配置 |
| `BM25_CONFIG` | BM25 索引文件路径和 Top-K |
| `RRF_CONFIG` | RRF 融合权重和常数 |
| `RERANKER_CONFIG` | 重排候选数和最终返回数 |
| `CHUNK_CONFIG` | 切片大小和重叠设置 |
